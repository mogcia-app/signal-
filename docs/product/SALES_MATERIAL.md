## 1. SNS運用代行のよくある課題

### 1.1 クライアントごとのブランド設定・トーン管理が大変

**課題の詳細**:
- クライアントごとに異なるブランド設定、口調、トーンを覚えて管理する必要がある
- 複数クライアントを同時に運用する際、ブランド設定を混同してしまう
- クライアントごとのNGワードや禁止構成を把握しきれない
- 新しい担当者が入った際、各クライアントの設定を引き継ぐのが大変

**よくある原因**:
- ブランド設定が属人的な知識として管理されている
- ツールやシステムで一元管理できていない
- クライアントごとの設定が文書化されていない
- 引き継ぎの仕組みが整っていない

### 1.2 複数クライアントの運用で効率化が難しい

**課題の詳細**:
- クライアントごとに投稿作成のプロセスが異なり、効率化できない
- 投稿作成に時間がかかり、クライアント数を増やせない
- コメント返信が手作業で、複数クライアントを同時に対応するのが大変
- 分析作業が手作業で、クライアント数に比例して工数が増える

**よくある原因**:
- クライアントごとにカスタマイズが必要で、テンプレート化できない
- 効率的なツールがない
- 自動化ができていない
- AIの活用が不十分

### 1.3 クライアントごとの成果を可視化・報告する負担

**課題の詳細**:
- 月次レポートの作成に時間がかかる
- クライアントごとに異なるKPIを追跡・報告する必要がある
- データ分析が手作業で、クライアント数に比例して工数が増える
- クライアントへの報告資料作成が大変

**よくある原因**:
- レポート作成ツールがない
- データ分析が手作業
- クライアントごとのKPI設定が統一されていない
- 自動レポート生成機能がない

### 1.4 属人的な運用ノウハウの課題

**課題の詳細**:
- 成功した投稿のパターンが担当者個人の知識として蓄積されている
- 過去のデータを活用できず、同じ失敗を繰り返してしまう
- チーム内でノウハウが共有されていない
- 担当者が変わると、クライアントの運用品質が低下する

**よくある原因**:
- データが蓄積されていない
- 分析ツールが不足している
- 成功パターンの可視化ができていない
- 学習メカニズムがない

### 1.5 スケーラビリティの課題

**課題の詳細**:
- クライアント数を増やそうとすると、人件費が比例して増える
- 1人あたりの担当クライアント数に限界がある
- クライアントごとのカスタマイズ対応が大変で、スケールできない
- 継続的な改善提案が手作業で、クライアント数に比例して工数が増える

**よくある原因**:
- 効率化の仕組みがない
- 自動化ができていない
- ツールやシステムがスケールに対応していない
- AIの活用が不十分

## 2. Signal. の解決アプローチ（AI × 人のクリエイティブ）

### 2.1 AIが学び続ける仕組み

**Signal.の独自性**:
- **MasterContext**: アカウント専用の学習データを継続蓄積
  - 投稿履歴、業種、ターゲット、トーン、NGワード、成功/失敗例
  - フィードバック、アクションログ
- **RAG（Retrieval-Augmented Generation）**: 過去すべての投稿を深く参照
  - 類似投稿の自動検出
  - 成功パターンの掘り起こし
  - 失敗パターンの回避

**効果**:
- 使えば使うほど精度が向上する「成長するAI」
- あなたのアカウント専用の最適化された提案
- ブランド設定・口調・世界観を反映した提案

### 2.2 AIが自動でPDCAを回し続ける

**Signal.の独自ロジック**:

```
Plan (計画)
├─ 目標設定・投稿シミュレーション
├─ AI戦略生成（MasterContext参照）
└─ 投稿計画まで自動生成

Do (実行)
├─ 投稿文・ハッシュタグ生成
├─ コメント返信アシスト
└─ 週次スケジュール生成

Check (分析)
├─ KPI分解とドリルダウン
├─ 投稿単位の詳細分析
├─ 成功パターンの自動検出
└─ 過去の投稿との比較分析

Act (改善)
├─ 改善案を自動生成
├─ 投稿計画まで自動生成
└─ 次のPlanに反映

Learn (学習) - 学習ダッシュボード
├─ AIとの学習状況を可視化
├─ 成功パターンと改善ポイントを自動抽出
├─ 学習進捗タイムライン
└─ フィードバック・アクション履歴
```

**効果**:
- 各フェーズの実行度を数値化して可視化
- "成果が出る方向に寄せ続ける"改善案を自動生成
- 継続的な改善サイクルが自動で回る
- 学習ダッシュボードで、AIとの学習状況を可視化し、成功パターンを次のPlanに反映

### 2.3 多角的で深い分析

**Signal.の分析力**:

**74項目以上のデータ統合分析**:
- KPI分解（20要素）: リーチ、保存、フォロワー、エンゲージメントをそれぞれ深掘り
- フィード統計（17項目）: いいね、コメント、シェア、保存、リーチ、フォロワー増加など
- リール統計（19項目）: 再生時間、スキップ率など
- オーディエンス（14項目）: 性別、年齢層別の分析
- 基本メトリクス（4項目）: いいね、コメント、シェア、閲覧数

**投稿単位の詳細分析**:
- 過去30件の投稿を個別に詳細分析
- クラスタ分析により成功パターン・失敗パターンを自動検出
- 各投稿のリーチ、エンゲージメント、閲覧ソース、オーディエンスまで分析

**過去の投稿との比較分析**:
- 投稿文、投稿時間、ハッシュタグを分析
- 直近の投稿や過去の投稿の詳細データを軸に比較
- どこが違ったのか、どこを伸ばせば改善につながるかを学習した上で次の一手を提案

**効果**:
- 表面的な情報ではなく、深い分析から具体的な改善案まで提示
- 成功パターンを次の投稿に反映
- PDCAサイクルで継続的に改善

### 2.4 具体的な投稿例まで提示

**一般的なAIツール**:
- 「エンゲージメントを高めましょう」
- 「リーチ数を増やすには投稿頻度を上げましょう」
- → 当たり前すぎて行動に移せない

**Signal.**:
- 成功パターンの自動検出
- 具体的な改善案（例: Before/After投稿を週2回に増やす）
- 具体的な投稿例（タイトル・キャプション・ハッシュタグ）
- 投稿タイミングまで具体的に提案
- → すぐに実行できる

### 2.5 高いカスタマイズ性

**Signal.のカスタマイズ機能**:
- ブランド設定
- 口調・トーン
- 世界観
- 禁止構成・NGワード
- CTA方針
- 業種・ターゲット

**効果**:
- すべてのAI提案にカスタマイズ設定が反映される
- ブランド一貫性を保った提案
- あなたのブランドに合わせた提案

### 2.6 複数クライアント管理の効率化・スケーラビリティ

**Signal.の運用代行向け機能**:

**クライアントごとのブランド設定を一元管理**:
- クライアントごとに独立したブランド設定を保存・管理
- 設定を切り替えるだけで、そのクライアント専用のAI提案を生成
- 新しい担当者でも、設定を確認するだけでクライアントの特性を把握可能
- 引き継ぎが簡単で、属人的な知識に依存しない

**複数アカウント対応**:
- プロフェッショナルプラン以上で、最大3アカウントまで対応
- エンタープライズプランでは無制限のアカウント管理が可能
- クライアントごとのデータが完全に分離され、混同のリスクがない

**自動レポート生成**:
- クライアントごとの月次レポートを自動生成
- クライアントごとのKPIを自動で追跡・可視化
- レポート作成の工数を大幅に削減（手作業の数時間 → 自動生成の数分）

**効率的な運用フロー**:
- 投稿作成時間を大幅に短縮（AI生成により、手作業の2時間 → 30分）
- コメント返信アシストにより、返信時間を短縮（平均2時間 → 30分）
- 1人で複数クライアントを効率的に運用可能

**効果**:
- クライアント数を増やしても、人件費が比例して増えない
- 1人あたりの担当クライアント数を増やせる
- スケーラブルな運用体制を実現
- クライアントごとの品質を維持しながら、効率化を実現

## 3. AIツール「Signal.」の概要

### 3.1 Signal.とは

**Signal.**は、Instagram運用を"分析 → 学習 → 改善"まで一気通貫で支援する、**AI主導のグロースハックツール**です。

運用者が入力する投稿データやフィードバック、AIが算出する指標を **MasterContext** として蓄積し、次の一手に直結するプランやコンテンツ生成までを自動化します。

### 3.2 主な機能

#### 3.2.1 月次レポート（`/instagram/report`）

**機能**:
- AIの自動解析で「今月のまとめ」「先月比較」「注目ポイント」「次のアクションプラン」をワンカードで提示
- PDCA実行度・振り返りデータ件数・改善反映率などPDCAメトリクスを表示
- リスク・異常検知と投稿タイプ別インサイトも独立カードで表示
- 過去AIサマリーの履歴取得ボタン。比較しながら改善を追跡可能

**特徴**:
- ボタン実行 or バックグラウンド取得
- ワンカードで全体像を把握
- 改善案を自動生成

#### 3.2.2 学習ダッシュボード（`/learning`）

**機能**:
- MasterContextに蓄積したフィードバック・アクションログを一覧化
- 学習バッジ、タイムライン、ポストパターン要約、成功/改善ギャラリーを表示
- Deep diveカードでクラスタ情報や派生指標、AIサマリーを確認可能
- Infoツールチップ完備で「なぜこの指標が重要か」を補足

**特徴**:
- AIとの学習状況を可視化
- 成功パターンと改善ポイントを自動抽出
- 継続的な改善をサポート

#### 3.2.3 投稿分析（`/analytics/feed`, `/instagram/analytics/reel`）

**機能**:
- 投稿分析フォーム＋統計カード＋AIインサイトの3カラム構成
- コメント＆返信ログ、満足度フィードバック、AIによる深掘り要約を連携
- AIインサイトはMasterContextと日次データを参照し、投稿時間・曜日・ハッシュタグまで分析
- 分析データのリセットボタンで重複データや誤入力を即クリア

**特徴**:
- 投稿単位の詳細分析
- AIインサイトで深掘り
- データ修正が可能

#### 3.2.4 投稿詳細（`/instagram/posts/[id]`）

**機能**:
- 投稿単位のディープダイブカード（クラスタ情報、比較差分、AIサマリータブ）
- 保存済みAIサマリーと最新生成の比較、アクションログ・フィードバック・コメントログを一画面で表示
- AIサマリーをサーバー側に保存し、分析履歴として活用

**特徴**:
- 投稿ごとの深掘り分析
- 過去の投稿との比較
- 分析履歴の保存

#### 3.2.5 投稿ラボ（`/instagram/lab/{feed|reel|story}`）

**機能**:
- 投稿エディタ + AIのテンプレート支援、構成生成、画像ヒント提案
- 右カラムに「コメント返信アシスト」「よく使う文言」「ハッシュタグ」セクションを配置しスクロール可能
- リール・ストーリー固有の構成生成APIとも連携
- コメントアシストは投稿コンテキストを渡し、ターン＆トーン別の返信案を生成

**特徴**:
- AIによる投稿生成支援
- コメント返信アシスト
- ブランド設定を反映

#### 3.2.6 運用計画（`/instagram/plan`）

**機能**:
- シミュレーション（投稿頻度/予算/目標変更など）と現行プラン表示
- AI戦略生成APIと連携しMasterContextを活用した提案が可能
- リアルな代替目標のワンボタン生成、異常検知からのシナリオ再提案にも対応予定

**特徴**:
- 目標達成シミュレーション
- AI戦略生成
- 現実可能性の自動判定

#### 3.2.7 KPI分解とドリルダウン（`/instagram/kpi`）

**機能**:
- KPI分解: リーチ、保存、フォロワー、エンゲージメントをそれぞれ深掘り
- ドリルダウン: 各KPIのセグメント内訳、貢献度の高い投稿、AIインサイト
- 日別KPI推移
- 時間帯 × コンテンツタイプヒートマップ
- ハッシュタグ分析
- オーディエンス構成サマリー

**特徴**:
- 多角的なKPI分析
- ドリルダウンで詳細を確認
- 可視化で直感的に理解

### 3.3 技術スタック

- **Frontend**: Next.js 15, React 19, TypeScript, Tailwind CSS
- **Backend**: Firebase Functions, Node.js
- **AI**: OpenAI GPT-4o-mini
- **Database**: Firebase Firestore
- **Authentication**: Firebase Authentication
- **Hosting**: Vercel
- **Monitoring**: Sentry

### 3.4 セキュリティ

- Firebase Authenticationによる認証
- Firestore Security Rulesによるデータアクセス制御
- APIレート制限による不正アクセス防止
- セキュリティログによる監視
- データ削除防止ロジック（重要データの保護）

## 4. サービス内容（プラン紹介）

### 4.1 プラン構成

Signal.は、お客様のニーズに合わせて柔軟なプランをご用意しています。

#### 4.1.1 ベーシックプラン

**対象**: 個人事業主、小規模事業者

**含まれる機能**:
- 月次レポート（AI自動解析）
- 学習ダッシュボード
- 投稿分析（フィード・リール）
- 投稿ラボ（AI生成支援）
- 運用計画（シミュレーション）
- KPI分解とドリルダウン

**制限**:
- AI生成回数: 月50回まで
- 分析データ保存期間: 6ヶ月
- サポート: メールサポートのみ

**料金**: お問い合わせください

#### 4.1.2 プロフェッショナルプラン

**対象**: 中小企業、複数アカウント運用

**含まれる機能**:
- ベーシックプランの全機能
- 複数アカウント対応（最大3アカウント）
- カスタムブランド設定
- 優先サポート
- 月次ミーティング（30分）

**制限**:
- AI生成回数: 月200回まで
- 分析データ保存期間: 12ヶ月
- サポート: メール + チャットサポート

**料金**: お問い合わせください

#### 4.1.3 エンタープライズプラン

**対象**: 大企業、代理店、複数ブランド運用

**含まれる機能**:
- プロフェッショナルプランの全機能
- 無制限アカウント対応
- 高度なカスタマイズ（API連携、独自機能開発）
- 専任サポート担当
- 月次ミーティング（60分）
- カスタムレポート

**制限**:
- AI生成回数: 無制限
- 分析データ保存期間: 無制限
- サポート: 専任担当による24時間サポート

**料金**: お問い合わせください

### 4.2 追加オプション

#### 4.2.1 AIカスタマイズ

- ブランド設定の詳細カスタマイズ
- 独自のトーン・世界観設定
- NGワード・禁止構成の設定
- CTA方針のカスタマイズ

#### 4.2.2 専任チーム

- 専任の運用サポートチーム
- 月次ミーティングの拡張
- カスタムレポートの作成
- 戦略立案のサポート

#### 4.2.3 データエクスポート

- 分析データのエクスポート
- カスタムレポートの作成
- API連携によるデータ取得

### 4.3 料金体系

**基本料金**:
- プランごとに月額固定料金

**従量課金**:
- AI生成回数がプラン上限を超えた場合
- 追加の分析データ保存期間

**初期費用**:
- セットアップ費用（初回のみ）
- オンボーディング費用（初回のみ）

**詳細な料金については、お問い合わせください。**

## 5. 運用開始までの流れ

### 5.1 ステップ1: お問い合わせ・ヒアリング

**期間**: 1-2週間

**内容**:
- お問い合わせフォームまたはメールでご連絡
- 初回ヒアリング（30-60分）
  - 現在のSNS運用状況
  - 課題・ニーズの確認
  - 目標の設定
- プラン提案・見積もり提示

### 5.2 ステップ2: 契約・セットアップ

**期間**: 1週間

**内容**:
- 契約書の締結
- 初期費用のお支払い
- アカウント作成
- 初期設定（ブランド設定、トーン、世界観など）

### 5.3 ステップ3: オンボーディング

**期間**: 2-4週間

**内容**:
- オンボーディングミーティング（60-90分）
  - ツールの使い方説明
  - 主要機能のデモ
  - よくある質問への回答
- 初期データの入力支援
  - 過去の投稿データの入力
  - フォロワー数の登録
  - 目標の設定
- サポートチャネルの案内

### 5.4 ステップ4: 運用開始

**期間**: 継続

**内容**:
- 通常運用の開始
- 月次ミーティング（プロフェッショナルプラン以上）
- サポート対応
- 継続的な改善

### 5.5 サポート体制

**メールサポート**:
- 営業時間内（平日10:00-18:00）に対応
- 回答まで24時間以内

**チャットサポート**（プロフェッショナルプラン以上）:
- 営業時間内に対応
- リアルタイムでの回答

**専任サポート**（エンタープライズプラン）:
- 24時間対応
- 専任担当によるサポート

## 6. AIレポート・分析サンプル

### 6.1 月次レポートサンプル

#### 6.1.1 今月のまとめ

```
【2024年1月のまとめ】
```

## 8. よくある質問（FAQ）

### 8.1 機能・使い方について

#### Q1. Signal.はどのような機能がありますか？

A. Signal.は、以下の主要機能を提供しています：
- 月次レポート（AI自動解析）
- 学習ダッシュボード（AIとの学習状況可視化）
- 投稿分析（フィード・リール）
- 投稿ラボ（AI生成支援、コメント返信アシスト）
- 運用計画（シミュレーション、AI戦略生成）
- KPI分解とドリルダウン

詳細は「[3. AIツール「Signal.」の概要](#3-aiツールsignalの概要)」をご覧ください。

#### Q2. AIが勝手に投稿しますか？

A. いいえ、Signal.は投稿を自動で実行することはありません。AIが提案する投稿案を確認していただき、ご自身で投稿を実行していただきます。投稿ラボで生成された投稿案をコピーして、Instagramに投稿していただく形になります。

#### Q3. 過去のデータがなくても大丈夫ですか？

A. はい、大丈夫です。Signal.は、データが蓄積されるにつれて精度が向上する「成長するAI」です。初期段階では、一般的なベストプラクティスに基づいた提案をしますが、データが蓄積されるにつれて、あなたのアカウント専用の最適化された提案をします。

#### Q4. ブランド設定はどのように反映されますか？

A. オンボーディング時に、ブランド設定（口調・トーン・世界観・NGワード・禁止構成・CTA方針など）を設定していただきます。その後、すべてのAI提案（投稿生成、コメント返信アシスト、戦略提案など）に、この設定が反映されます。

#### Q5. 複数のアカウントを管理できますか？

A. はい、プロフェッショナルプラン以上で、複数アカウントの管理が可能です。エンタープライズプランでは、無制限のアカウント管理が可能です。

### 8.2 料金・プランについて

#### Q6. 料金はいくらですか？

A. プランによって異なります。詳細な料金については、お問い合わせください。基本料金は月額固定料金で、AI生成回数がプラン上限を超えた場合に従量課金が発生します。

#### Q7. 初期費用はかかりますか？

A. はい、セットアップ費用とオンボーディング費用が初回のみ発生します。詳細はお問い合わせください。

#### Q8. 解約はできますか？

A. はい、いつでも解約可能です。解約手続きについては、サポートまでお問い合わせください。

### 8.3 セキュリティ・データについて

#### Q9. データは安全ですか？

A. はい、Signal.は以下のセキュリティ対策を実施しています：
- Firebase Authenticationによる認証
- Firestore Security Rulesによるデータアクセス制御
- APIレート制限による不正アクセス防止
- セキュリティログによる監視
- データ削除防止ロジック（重要データの保護）

詳細は「[3.4 セキュリティ](#34-セキュリティ)」をご覧ください。

#### Q10. データはどこに保存されますか？

A. データは、Firebase Firestore（Google Cloud Platform）に保存されます。データは暗号化され、セキュアに管理されています。

#### Q11. データをエクスポートできますか？

A. はい、データエクスポートオプション（有料）で、分析データのエクスポートが可能です。API連携によるデータ取得も可能です。

### 8.4 サポートについて

#### Q12. サポートはどのように受けられますか？

A. プランによって異なります：
- ベーシックプラン: メールサポート（営業時間内、24時間以内に回答）
- プロフェッショナルプラン: メール + チャットサポート（営業時間内）
- エンタープライズプラン: 専任サポート（24時間対応）

#### Q13. 月次ミーティングはありますか？

A. プロフェッショナルプラン以上で、月次ミーティング（30-60分）を実施しています。運用状況の確認、改善提案、質問への回答などを行います。

### 8.5 その他

#### Q14. 他社ツールとの違いは何ですか？

A. Signal.の主な差別化ポイントは以下の通りです：
- **AIが学び続ける仕組み**: MasterContext + RAGにより、使えば使うほど精度が向上
- **AIが自動でPDCAを回し続ける**: 継続的な改善サイクルが自動で回る
- **多角的で深い分析**: 74項目以上のデータ統合分析、投稿単位の詳細分析
- **具体的な投稿例まで提示**: 抽象的なアドバイスではなく、すぐに実行できる具体的な提案

詳細は「[2. Signal. の解決アプローチ（AI × 人のクリエイティブ）](#2-signal-の解決アプローチai--人のクリエイティブ)」をご覧ください。

#### Q15. 導入までの期間はどのくらいですか？

A. お問い合わせから運用開始まで、通常2-4週間程度です。詳細は「[5. 運用開始までの流れ](#5-運用開始までの流れ)」をご覧ください。

#### Q16. デモはできますか？

A. はい、デモのご依頼を承っております。お問い合わせフォームまたはメールでご連絡ください。
