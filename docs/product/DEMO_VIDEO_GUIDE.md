# Signal. デモ動画作成ガイド

## 🎬 デモ動画の目的と効果

### 目的

- **視覚的な理解**: 文章だけでは伝わりにくい機能を実際に見せる
- **信頼性の向上**: 実際の画面を見せることで信頼を獲得
- **コンバージョン率向上**: 動画があると購入率が80%以上アップ
- **説明コスト削減**: 営業時間の短縮、効率化

### 理想的な動画の長さ

- **HP用**: 30秒〜2分（短く印象的）
- **詳細版**: 3〜5分（機能説明）
- **完全版**: 10分以内（全体像の理解）

## 🛠️ 動画作成方法

### 方法1: 画面録画ツールを使用（推奨）

#### おすすめツール

##### **macOS（Mac）**

1. **QuickTime Player（無料・標準装備）**

   ```
   使い方：
   1. QuickTime Playerを開く
   2. 「ファイル」→「新規画面収録」
   3. 録画開始ボタンをクリック
   4. 録画したい領域を選択
   5. 停止ボタンで終了
   ```

   - ✅ 無料・標準装備

   - ❌ 編集機能が限定的

2. **OBS Studio（無料・高機能）**

   ```
   1. シーンに「ウィンドウキャプチャ」を追加
   2. Signal.の画面を選択

   4. 録画終了ボタンで停止
   ```

   - ✅ 完全無料
   - ✅ 高画質
   - ✅ 編集機能も充実
   - ❌ 設定がやや複雑

## 📝 動画作成の手順

### ステップ1: 事前準備

#### 1. シナリオ作成

```
【書き出すこと】
- 何を伝えたいか（目的）
- どんな流れで見せるか（構成）
- 各シーンの時間配分
- 説明するセリフやテキスト
```

#### 2. 素材の準備

- 実際のSignal.アカウント（デモ用）
- サンプルデータ（投稿、分析結果など）
- スクリプト（話す内容）
- BGM（必要に応じて）

#### 3. 環境設定

- ブラウザを最新バージョンに
- 不要な通知をOFF
- きれいな画面（不要なウィンドウを閉じる）
- 解像度を統一（1920x1080推奨）

---

### ステップ2: 録画

#### 録画のコツ

1. **ゆっくり操作する**
   - 早すぎると見づらい
   - 1つの操作につき2-3秒ほど

2. **マウスカーソルを見やすく**
   - マウスカーソルを大きく（設定で変更）
   - クリック時にハイライト表示（ツールによって可能）

3. **音声解説を入れる**
   - 操作しながら説明
   - または、後で音声を追加

4. **複数回録画する**
   - 1回で完璧を目指さない
   - 良い部分だけ切り出して使う

---

### ステップ3: 編集

#### 編集のポイント

1. **不要な部分を削除**
   - 操作の待ち時間
   - 失敗した部分
   - 余計なクリック

2. **テキストを追加**

   ```
   例：
   - 「AIが最適な投稿を提案」
   - 「90%自動化で時間削減」
   - 「30日間無料トライアル」
   ```

3. **BGMを追加**
   - 明るく前向きな音楽
   - ボリュームは控えめに
   - 無料のBGMサイトを使用
     - YouTube Audio Library
     - Freesound
     - AudioJungle（有料）

4. **トランジション追加**
   - シーン切り替え時の効果
   - フェードイン/アウト
   - ただし、使いすぎない

5. **音声を追加/編集**
   - ナレーションを録音
   - 不要な雑音を削除
   - 音量を調整

---

### ステップ4: エクスポート・最適化

#### 推奨設定

```
【解像度】
- 1920x1080（Full HD）
- または 1280x720（HD）

【形式】
- MP4（最も互換性が高い）

【フレームレート】
- 30fps（標準）
- または 60fps（滑らか）

【ビットレート】
- 5-10 Mbps（高画質）
- または 2-5 Mbps（ファイルサイズ重視）
```

#### ファイルサイズの最適化

- YouTube: 自動で最適化される
- HP埋め込み: 5MB以下を推奨
- 大きすぎる場合は圧縮ツールを使用

## ⚠️ 注意点

### 1. 個人情報の取り扱い

- デモデータを使用（実名・実データは使わない）
- お客様のデータは絶対に使わない
- テストアカウントを用意

### 2. 権利関係

- BGMは商用利用可能なものを使用
- 画像・イラストも商用利用可能なものを使用

### 3. 動画の最適化

- ファイルサイズに注意（HP埋め込みは軽く）
- 読み込み速度を考慮

### 4. アクセシビリティ

- 字幕を追加（YouTubeの自動字幕機能）
- 音声だけでなくテキストでも情報を伝える

- 最初の5秒で引き込む

- CTAは最後だけでなく、途中にも配置
