# Signal. 投稿回数別プラン設計（松竹梅）

## 📊 プラン設計の基本方針

### 重要なポイント

1. **投稿回数は目安**: 実際の価値は「時間削減」と「PDCA自動化」
2. **使えば使うほど価値が出る**: 投稿回数が増えても作業時間はほぼ変わらない
3. **機能は全プラン共通**: Signal.の本質的価値（PDCA自動化）はすべてのプランで提供

---

## 💡 重要なメッセージ

### 「投稿回数に応じたプラン」の真意

```
「投稿回数に応じたプラン」という表記は、
お客様の運用頻度に合わせて料金を選択していただくためのものです。

しかし、Signal.の本質的な価値は「投稿回数」ではなく、
「時間削減」と「PDCA自動化」にあります。

投稿回数が増えても、Signal.を使えば作業時間はほぼ変わりません。
これが、従来の手作業運用とSignal.の決定的な違いです。
```

### 全プラン共通の価値

**すべてのプランで以下が無制限利用可能**:
1. **データ分析・KPI分解**: 自動実行
2. **月次レポート自動生成**: 自動実行
3. **AI診断・シミュレーション**: 無制限
4. **コメント返信アシスト**: 無制限
5. **投稿生成機能**: 無制限（想定投稿数は目安）

**これがSignal.の本質です。**
- フロントもバックもAIが常時動いている
- PDCAサイクルが自動で回る
- 投稿回数が増えても作業時間は変わらない

3. **全プランでPDCA自動化が無制限**
   - データ分析・月次レポート・AI診断がすべて自動
   - 使えば使うほど価値が出る設計
