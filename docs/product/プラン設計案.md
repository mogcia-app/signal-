# Signal. プラン設計案

## 現在の機能整理

### 📋 主要機能カテゴリ

#### 1. **投稿作成・編集**

**目的**: AIの力を借りて、高品質なコンテンツを効率的に作成する

**価値**:
- 投稿のアイデア出しから作成までを一貫してサポート
- 過去の投稿データや成功パターンを参照して、最適なコンテンツを提案
- A/Bテストを登録することで、データに基づいた改善が可能
- スナップショットインサイトで現在のアカウント状況を考慮した提案

**解決する課題**:
- 「何を投稿すればいいか分からない」
- 「投稿を作るのに時間がかかる」
- 「過去の投稿を参考にしたいが、どこを見ればいいか分からない」
- 「効果的な投稿パターンが分からない」

**機能詳細**:
- 🧪 投稿ラボ（Lab）
  - フィード/リール/ストーリー作成
    - タイトル、本文、ハッシュタグ（フィード・リールは最大5個）を入力
    - 画像・動画をアップロード（ドラッグ&ドロップまたはファイル選択）
    - 投稿のプレビュー表示で見た目を確認
    - 投稿を保存（下書き状態）または公開予約日時を設定
    - 既存投稿の編集（画像の後から追加も可能）
  - AI投稿生成（AI提案ハイライト）
    - プロンプト（例：「お店の雰囲気」「スタッフ紹介」）を入力してAIに投稿案を生成依頼
    - AIが生成したタイトル・本文・ハッシュタグの提案を確認
    - 提案内容をそのまま採用するか、手動で編集して調整
    - 過去の投稿データや成功パターンを参照した提案を受けられる
  - スナップショットインサイト
    - 現在のアカウント状況（フォロワー数、エンゲージメント率など）を参照
    - 過去の投稿傾向や成功パターンを表示
    - インサイトを参考にしながら投稿内容を調整
  - A/Bテスト登録
    - 投稿を作成する際に、A/Bテストとして登録可能
    - テストの仮説や想定KPIを記録
    - 複数の投稿パターンを比較検証できるよう準備
  - マスタコンテキスト参照
    - 過去の投稿データや学習した成功パターンを参照
    - AI生成時に自動的に参照され、より効果的な提案を受けられる

#### 2. **投稿管理**

**目的**: 作成した投稿を一元管理し、後から編集・改善できるようにする

**価値**:
- 全投稿を一覧で確認でき、過去の投稿を振り返りやすい
- 投稿タイプやステータスでフィルタリング可能
- 公開後に画像を追加・編集できる柔軟性
- 投稿の詳細を確認し、必要に応じて削除できる

**解決する課題**:
- 「過去に何を投稿したか分からない」
- 「投稿した後、画像を追加したいができない」
- 「投稿を管理する場所がない」
- 「投稿の履歴を確認したい」

**機能詳細**:
- 📚 投稿一覧
  - 投稿の表示・フィルタリング
    - 全投稿をカード形式で一覧表示（タイトル、画像、ステータス、投稿タイプ）
    - 投稿タイプ（フィード/リール/ストーリー）でフィルタリング
    - ステータス（下書き/公開済み/予約投稿）でフィルタリング
    - 日付順・更新順などでソート
  - 投稿詳細・編集
    - 各投稿カードをクリックして詳細ページを表示
    - 投稿の全情報（タイトル、本文、ハッシュタグ、画像、作成日時など）を確認
    - 詳細ページから編集画面に遷移して内容を修正
    - 編集後は保存して更新
  - 投稿削除
    - 投稿一覧または詳細ページから削除ボタンをクリック
    - 確認ダイアログで削除を確定
    - 削除後は一覧から除外される
  - 画像の後から追加
    - 既に作成済みの投稿（画像なし）に後から画像を追加
    - 編集画面で画像アップロード機能を使用
    - Base64形式で保存され、後からでも画像を表示・管理可能

#### 3. **投稿分析**

**目的**: 個別の投稿のパフォーマンスを詳細に分析し、改善のヒントを得る

**価値**:
- 各投稿の実績データ（いいね、リーチ、保存数など）を入力・管理
- AIが投稿の特徴を分析し、成功・失敗の要因を自動抽出
- コメントや返信のログを管理し、エンゲージメントを追跡
- 満足度フィードバックを記録し、主観的な評価も考慮

**解決する課題**:
- 「投稿の成果を把握したいが、どこに記録すればいいか分からない」
- 「なぜこの投稿が伸びた（伸びなかった）のか分からない」
- 「投稿ごとの詳細な分析ができない」
- 「過去の投稿の成功パターンを見つけたい」

**機能詳細**:
- 📊 投稿分析（Analytics）
  - 実績データ入力
    - 各投稿の実績データを手動で入力
    - いいね数、リーチ数、保存数、コメント数などの数値を入力
    - オーディエンスデータ（性別、年代別の構成比）を入力
    - データは投稿ごとに保存され、後から参照・編集可能
  - 統計カード
    - 入力した実績データを元に、統計情報をカード形式で表示
    - エンゲージメント率、リーチ率などの指標を自動計算
    - グラフやチャートで視覚的に表示
    - 過去の投稿との比較も可能
  - AIインサイト（個別投稿の深掘り分析）
    - 投稿の実績データと内容を元に、AIが分析を実行
    - 投稿が成功した（または失敗した）要因を自動抽出
    - ハッシュタグ、投稿時間、コンテンツ内容などの観点から分析
    - 改善のヒントや次のアクションを提案
    - 分析結果は保存され、後から参照可能
  - コメント＆返信ログ
    - 投稿に付いたコメントを記録・管理
    - 返信した内容も記録
    - エンゲージメントの質を追跡
    - コメントの傾向や頻出キーワードを把握
  - 満足度フィードバック
    - 投稿に対する主観的な評価を記録（例：5段階評価）
    - 自分なりの成功・失敗の判断を保存
    - AI分析と組み合わせて、より精度の高い学習データとして活用

#### 4. **レポート・KPI**

**目的**: 月次や全体のパフォーマンスを可視化し、戦略的な判断を支援する

**価値**:
- 月次の投稿実績を自動集計し、AIが要点をまとめて提示
- 前月との比較で成長を実感できる
- 次のアクションプランを自動生成し、具体的な改善案を提示
- KPIをグラフやヒートマップで可視化し、傾向を把握
- PDCAメトリクスで運用サイクルの健全性を測定

**解決する課題**:
- 「月次で成果をまとめたいが、時間がかかる」
- 「全体の傾向が分からない」
- 「次の月に何をすべきか分からない」
- 「PDCAサイクルが回せているか分からない」
- 「どの時間帯に投稿するのが効果的か分からない」

**機能詳細**:
- 📈 月次レポート
  - AI自動解析
    - ボタンクリックまたは自動実行で、月次の投稿データをAIが解析
    - 全投稿の実績データを集計し、傾向を分析
    - 解析結果は「今月のまとめ」として自動生成
  - 今月のまとめ・先月比較
    - 今月の投稿数、総いいね数、総リーチ数などの主要指標を表示
    - 先月のデータと比較して、成長率や変化を表示
    - グラフや数値で視覚的に成長を実感
    - 注目すべき投稿や傾向をハイライト
  - 次のアクションプラン
    - AIが分析結果を元に、翌月の改善アクションを自動提案
    - 具体的なアクション項目をリスト化（例：「ハッシュタグを5個に統一する」「投稿時間を午後に集中させる」）
    - チェックボックスでアクションを選択し、学習ダッシュボードに記録
    - 実行可能な具体的な改善案を提示
  - PDCAメトリクス
    - PDCAサイクルの健全性を測定（Plan: 計画、Do: 実行、Check: 評価、Act: 改善）
    - PDCA実行度、振り返りデータ件数、改善反映率などの指標を表示
    - 運用サイクルが適切に回っているかを数値で可視化
- 🎯 KPIコンソール
  - KPI可視化
    - 主要なKPI（フォロワー数、エンゲージメント率、リーチ数など）をダッシュボード形式で表示
    - 期間を選択して、時系列での変化をグラフで確認
    - 目標値との比較も可能
    - 複数のKPIを一度に確認できる
  - タイムスロットヒートマップ
    - 投稿時間帯と曜日をマトリクス形式（ヒートマップ）で表示
    - どの時間帯・曜日の投稿が効果的だったかを色の濃淡で視覚化
    - 効果的な投稿タイミングを一目で把握
    - 投稿スケジュールの最適化に活用
  - オーディエンス分析
    - フォロワーの属性（性別、年代、地域など）を可視化
    - 円グラフや棒グラフで構成比を表示
    - オーディエンスの変化を追跡
    - ターゲット層にリーチできているかを確認

#### 5. **学習・最適化**

**目的**: 過去のデータから継続的に学習し、運用を自動的に最適化する

**価値**:
- MasterContextに蓄積されたデータから成功パターンを自動抽出
- 学習バッジやタイムラインで学習状況を可視化
- 運用計画のシミュレーションで、目標達成の可能性を事前に確認
- AI診断で現在の運用状況を評価し、改善点を指摘
- AI戦略生成で、目標に応じた最適な戦略を自動提案

**解決する課題**:
- 「過去のデータを活かしきれていない」
- 「目標設定が現実的かどうか分からない」
- 「運用計画を立てたいが、どうすればいいか分からない」
- 「継続的に改善している実感がない」
- 「成功パターンを体系的に学習したい」

**機能詳細**:
- 🗂️ 学習ダッシュボード
  - MasterContextの蓄積・可視化
    - 過去の投稿データ、分析結果、フィードバックなど、すべての学習データを「MasterContext」として蓄積
    - 蓄積されたデータの量や質を可視化（タイムライン形式など）
    - データが増えるほど、AIの提案精度が向上
    - 蓄積状況をグラフや指標で確認
  - 学習バッジ
    - 特定の条件を達成すると、学習バッジが付与される（例：「100投稿達成」「エンゲージメント率向上」）
    - バッジ一覧で学習の進捗を確認
    - 継続的な改善のモチベーション向上
    - 達成感を可視化
  - 成功パターンの抽出
    - AIが過去の投稿データから成功パターンを自動抽出
    - 「このハッシュタグの組み合わせが効果的」「この時間帯の投稿が伸びやすい」などのパターンを表示
    - 成功パターンに基づいた投稿提案を受けられる
    - パターンを参照しながら、新しい投稿を作成
  - 改善ギャラリー
    - 過去の改善アクションとその結果をギャラリー形式で表示
    - どの改善が効果的だったかを視覚的に確認
    - 改善の軌跡を振り返り、継続的な学習に活用
- 📋 運用計画（Plan）
  - 目標設定シミュレーション
    - フォロワー数やエンゲージメント率などの目標を設定
    - AIが目標達成の現実可能性を5段階で評価（非常に現実的〜非常に困難）
    - 目標達成に必要な投稿数や投稿ペースを自動計算
    - 成長曲線をグラフで可視化し、達成予測日を表示
    - 困難な場合、現実的な代替案を自動提案
  - AI診断
    - 現在の運用計画や投稿データをAIが診断
    - 運用の強み・弱み、改善点を自動抽出
    - 診断結果をレポート形式で表示
    - 具体的な改善提案を受けられる
  - AI戦略生成
    - 設定した目標や診断結果を元に、AIが最適な運用戦略を自動生成
    - 複数の戦略案を提示（例：リーチ拡大戦略、エンゲージメント向上戦略）
    - 各戦略の特徴や期待効果を説明
    - 戦略を選択して、投稿作成や運用計画に反映

## 💡 プラン設計の提案

### 案1: ユーザーの提案（シンプル）⭐ **ベース案**

| プラン | 価格 | 機能 |
|--------|------|------|
| **梅** | ¥15,000/月 | 投稿ラボのみ |
| **竹** | ¥30,000/月 | 投稿ラボ + 投稿一覧 |
| **松** | ¥60,000/月 | 全機能 |

**メリット:**
- シンプルで分かりやすい
- 価格差が明確（2倍ずつ）
- **アップセル動機が明確**（投稿を作成した後、管理・分析したくなる）
- 梅プランは「作る」ことに集中、シンプル

**戦略的意図:**
- 梅プランで投稿を作成
- 作成した投稿を「管理したい」「分析したい」という欲求が自然と生まれる
- 竹プランへのアップセルが自然な流れになる

## 🎯 推奨案: **案1（ユーザー提案）** をベースに、**案2（機能段階型）** で竹・松の機能を明確化

1. **段階的な価値提供**
   - 梅: 最低限の作成機能（個人・小規模事業者向け）

   - 松: 継続学習・最適化で差別化（企業・本格的運用者向け）

2. **アップグレード動機が明確**

   - 竹→松: 「分析結果を活かして継続改善したい」

3. **価格設定が妥当**
   - 竹プランで投稿分析と月次レポートが入ることで、¥30,000の価値が明確
   - 松プランで学習・最適化機能が入ることで、¥60,000の価値が明確

## 💭 検討ポイント

### 1. 梅プランは「作成に特化」戦略

**方針: 投稿ラボのみ**
- 投稿作成に集中できるシンプルな体験
- 作成した投稿の管理・分析はできない
- **「作った投稿を管理したい」という欲求が自然に生まれる**
