# Signal. プロダクト概要（LP向け）

## 1. プロダクトの全体像
Signal. は Instagram 運用を“分析 → 学習 → 改善”まで一気通貫で支援する、AI主導のグロースハックツールです。  
運用者が入力する投稿データやフィードバック、AIが算出する指標を MasterContext として蓄積し、次の一手に直結するプランやコンテンツ生成までを自動化します。

**できることのハイライト**
- KPI・PDCAに基づく月次レポートと改善提案
- 投稿単位の深堀り分析と AI サマリー保存
- 学習ダッシュボードによる成功パターンの可視化
- 投稿ラボでの AI 生成・構成支援とコメント返信アシスト
- フィード／リール分析の統計・AIインサイト・学習ログ連携
- 行動ログ（フィードバック・アクション）を使った MasterContext の継続学習

## 2. 主なページ / 機能別の特徴
### `/instagram/monthly-report` 月次レポート
- AI の自動解析（ボタン実行 or バックグラウンド取得）で、
  - 「今月のまとめ」「先月比較」「注目ポイント」「次のアクションプラン」「運用計画の振り返り」をワンカードで提示
  - PDCA実行度・振り返りデータ件数・改善反映率など PDCAメトリクスを表示
- リスク・異常検知と投稿タイプ別インサイトも独立カードで表示
- 過去 AI サマリーの履歴取得ボタン。比較しながら改善を追跡可能。

### `/learning` 学習ダッシュボード
- MasterContext に蓄積したフィードバック・アクションログを一覧化
- 学習バッジ、タイムライン、ポストパターン要約、成功/改善ギャラリーを表示
- Deep dive カードでクラスタ情報や派生指標、AIサマリーを確認可能
- Info ツールチップ完備で「なぜこの指標が重要か」を補足

### `/analytics/feed` / `/instagram/analytics/reel`
- 投稿分析フォーム＋統計カード＋AIインサイトの 3 カラム構成
- コメント＆返信ログ、満足度フィードバック、AIによる深掘り要約を連携
- AIインサイトは MasterContext と日次データを参照し、投稿時間・曜日・ハッシュタグまで分析
- 分析データのリセットボタンで重複データや誤入力を即クリア
- リール向けには `targetCategory="reel"` でプロンプトを特化

### `/instagram/posts/[id]`
- 投稿単位のディープダイブカード（クラスタ情報、比較差分、AIサマリータブ）
- 保存済み AI サマリーと最新生成の比較、アクションログ・フィードバック・コメントログを一画面で表示
- AIサマリーをサーバー側に保存し、分析履歴として活用

### `/instagram/lab/{feed|reel|story}`
- 投稿エディタ + AI のテンプレート支援、構成生成、画像ヒント提案
- 右カラムに「コメント返信アシスト」「よく使う文言」「ハッシュタグ」セクションを配置しスクロール可能
- リール・ストーリー固有の構成生成 API とも連携
- コメントアシストは投稿コンテキストを渡し、ターン＆トーン別の返信案を生成

### `/instagram/plan`
- シミュレーション（投稿頻度/予算/目標変更など）と現行プラン表示
- AI 戦略生成 API と連携し MasterContext を活用した提案が可能
- リアルな代替目標のワンボタン生成、異常検知からのシナリオ再提案にも対応予定

## 3. AI 学習とマスターコンテキスト
- Firestore に以下の学習ログを保存し MasterContext を更新  
  - `ai_post_feedback`, `ai_action_logs`, `ai_post_summaries`, `ai_overview_history` など
- MasterContext は `ragHitRate`, `learningPhase`, `feedbackStats`, `actionStats`, `postPatterns`, `pdcaMetrics`, `timeline` などを保持
- 取得した MasterContext は月次レポートやフィードAIインサイト、投稿生成プロンプトに流用
- 将来的にはベクトル DB を導入して高度な RAG（類似投稿の引き戻し）を検討

### Signal. における PDCA 循環の実体
| フェーズ | ツール内の主要セクション | 具体的な動き |
| --- | --- | --- |
| **Plan** | `/instagram/plan`、`/instagram/lab/{feed|reel|story}` | 目標設定・投稿シミュレーション・AI生成案の採用。MasterContextの投稿パターンや学習済みトーンをプロンプトに反映。 |
| **Do** | `/instagram/lab/*`、Zapsなどの外部投稿導線 | AI生成コンテンツをベースに運用者が撮影/投稿実行。コメント返信アシストでファン対話を効率化。 |
| **Check** | `/instagram/monthly-report`、`/analytics/feed`、`/analytics/reel`、`/instagram/posts/[id]` | KPI評価、PDCAメトリクス、ポストタイプ別インサイト、Deep Diveカードで実績を数値/定性の両面から検証。MasterContextが最新分析を取り込み。 |
| **Act** | `/learning` ダッシュボード、`/instagram/plan` | 学習バッジ・タイムライン・postPatternsから成功/改善ポイントを抽出し、次のPlanに反映。AI Action Logs ボタンで実行可否を記録し MasterContext を更新。 |

**ポイント**
- フィードバックやアクションログを保存すると MasterContext の `feedbackStats`/`actionStats`/`ragHitRate` が更新され、月次レポ―トやAI生成時のプロンプトに反映。
- `/analytics/*` でリセットや修正を行うと重複データが除去され、PDCAの検証がクリアに保たれる。
- `/instagram/posts/[id]` で各投稿の AI サマリーを保存 → `/learning` タイムラインで改善履歴として活用 → 次の投稿生成時に再利用、という循環が成立。

## 4. 他社ツールとの差別化ポイント
| 項目 | Signal. | 一般的なレポートツール |
| --- | --- | --- |
| **学習ループ** | フィードバック/アクションログを AI が継続蓄積し次の提案に活用 | 分析結果の閲覧のみで、ユーザーの行動までは学習しない |
| **AI ディープダイブ** | 投稿単位でクラスタ比較、差分指標、AIサマリーを生成・保存 | 投稿単位の深掘りは手作業 or 過去ログに残らないケースが多い |
| **PDCA 可視化** | PDCA実行度、振り返り件数、改善反映率をカード表示 | フォロワー数やエンゲージメント等の数値表示で終わる場合が多い |
| **コメント対応支援** | ラボ右カラムで AI がトーン別の返信案を即生成 | コメント返信は運用者が個別対応するのが一般的 |
| **MasterContext** | ユーザーアカウントごとの学習状態を API 全体で共有 | 分析レポートは単発で、生成機能と連携しないケースが多い |
| **データ修正性** | 分析データリセットや補正スクリプトでデータ整合性を維持 | 重複や誤入力の修正機能が限定的 |

## 5. LP 訴求のヒント
- 「AI と一緒に運用改善を回す」ことをメインコピーに  
- スクリーンショットは以下の流れで並べると効果的  
  1. 月次レポート（AIサマリー + PDCA指標 + リスク/インサイト）  
  2. 学習ダッシュボード（バッジ・タイムライン・パターン）  
  3. 投稿詳細（Deep dive + AIサマリー）  
  4. ラボ（AI生成＋コメント返信アシスト）  
  5. Analytics ページ（統計 + AI インサイト + リセット機能）
- ユーザーの声パートで「AI の提案で○○を実行」「フィードバックを入力するだけで改善アイデアが出る」など、学習ループの価値を強調

## 7. LP再構成プラン（現行LPの課題と改善方針）
### 現状の課題（[sgnalapp.com](https://www.sgnalapp.com/) 現行LPより）
- 「SNS運用90％自動化」などのキャッチはあるが、**ツール画面やユーザー体験が具体的に描かれていない**。
- 他社比較は抽象的で、**MasterContextやコメント返信アシストといった固有機能が伝わっていない**。
- ストーリーが「課題列挙 → ハイブリッド訴求 → 価格」で終わり、**“分析→学習→改善→生成”の循環が伝わっていない**。

### 新LPの構成アイデア
1. **ヒーローセクション**  
   - キャッチ例「AIが学び続けるSNSグロースOS」  
   - 月次レポートや学習ダッシュボードなど主要UIを並べたキービジュアル  
   - PDCA実行度やコメント対応時間など具体的な成果指標を提示
2. **課題→解決ストーリー**  
   - 既存の「投稿が伸びない」「ノウハウが分散」などの壁に対し、対応する機能を画面キャプチャ付きで紹介  
   - 例: PostTypeInsights＋Deep Diveカード = 「勝ちパターンが見えない」課題を解消
3. **プロダクトツアー**  
   - 本ドキュメント第2章の順序で各ページを紹介（monthly-report → learning → posts detail → analytics → lab → plan）  
   - 「何が表示され、どのボタンを押すとどうなるか」を短文で説明
4. **AI学習メカニズム**  
   - MasterContextの更新ループ（フィードバック→学習→提案）を図解  
   - 保存されるデータ例（ai_post_feedback等）と、どのプロンプトに反映されるかを記載
5. **比較表アップデート**  
   - 「テンプレAI」「運用代行」「Signal.」の比較を再構築し、学習ループやコメント支援などの差異を明示
6. **成果事例・数値**  
   - Betaユーザーや自社検証で得られた数値（PDCA実行度＋◯pt、改善反映率◯％など）を配置  
   - 可能なら導入前後のグラフやコメント評価のスクリーンショットを掲載
7. **導入フロー & 伴走体制**  
   - AIカスタマイズと専任チームの役割を図で示し、「AIが動かし、人が磨く」モデルを再強調  
   - 月次ミーティングやサポートチャネルを明記
8. **FAQ / CTA**  
   - 「AIが勝手に投稿する？」「過去データがなくても大丈夫？」など導入前の不安に回答  
   - 各セクション末に問い合わせ/デモ申込ボタンを配置

### 差別化を一言で伝えるコピー案
- 「分析も生成も、“御社専用AI” が学習して提案」
- 「フィードバックを入れるたびに成長。PDCAが止まらないSNS運用へ」
- 「コメント対応から次月の戦略まで、AIが提案し人が磨くグロースOS」
