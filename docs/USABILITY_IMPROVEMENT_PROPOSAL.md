# 初見ユーザー向け使いやすさ改善提案

作成日: 2026-02-07

## 背景

初見ユーザーから「ツール全体が使いづらい」というフィードバックがありました。改善案をまとめます。

---

## 現状の問題点

### 1. ナビゲーションの不明確さ
- サイドバーに多くのメニューがあるが、どこから始めればいいか分からない
- 各ページの役割が明確でない
- 次のステップが分からない

### 2. 機能の説明不足
- 各機能が何をするのか説明がない
- 専門用語（KPI、エンゲージメント率など）の説明がない
- ボタンや操作の説明がない

### 3. 初回ログイン後のガイダンス不足
- オンボーディングページはあるが、初回ログイン後の流れが分からない
- 最初に何をすべきかが明確でない

### 4. 空状態での説明不足
- データがない場合の説明が不十分
- 次のアクションが分からない

---

## 改善提案

### 提案1: 初回ログイン時のウェルカムツアー

**実装内容:**
- 初回ログイン時に自動で表示されるウェルカムモーダル
- 主要な機能を3-5ステップで紹介
- 「スキップ」ボタンで閉じられる
- 後で「ヘルプ」から再表示可能

**表示内容:**
1. ホームページの説明
2. 運用計画の重要性
3. 投稿ラボの使い方
4. 分析・レポートの見方

**実装優先度:** 高

---

### 提案2: 各ページに「このページについて」セクション

**実装内容:**
- 各ページの上部に折りたたみ可能な説明セクションを追加
- 「このページについて」ボタンで展開/折りたたみ
- 初回訪問時のみ自動で展開（localStorageで管理）

**例:**
```
┌─────────────────────────────────────┐
│ 📋 運用計画                          │
│ ▼ このページについて                │
├─────────────────────────────────────┤
│ 運用計画ページでは、今月の目標や    │
│ 戦略を設定できます。AIがこの計画を  │
│ 基に投稿文を生成します。            │
│                                     │
│ 【次のステップ】                    │
│ 1. 目標を設定する                   │
│ 2. 戦略を選択する                   │
│ 3. AIに計画を生成してもらう         │
└─────────────────────────────────────┘
```

**実装優先度:** 高

---

### 提案3: ツールチップの拡充

**実装内容:**
- 既存の`InfoTooltip`コンポーネントを活用
- 専門用語や操作にツールチップを追加
- ホバーまたはクリックで説明を表示

**追加箇所:**
- KPIコンソールの各指標
- エンゲージメント率などの専門用語
- ボタンや操作の説明
- フォーム項目の説明

**実装優先度:** 中

---

### 提案4: ホームページに「クイックスタート」セクション

**実装内容:**
- ホームページの上部に「クイックスタート」セクションを追加
- 初見ユーザー向けの3ステップガイド
- 各ステップにリンクボタンを配置

**表示内容:**
```
┌─────────────────────────────────────┐
│ 🚀 クイックスタート                  │
├─────────────────────────────────────┤
│ 1. 運用計画を作成する                │
│    → 運用計画ページへ                │
│                                     │
│ 2. 投稿文を生成する                  │
│    → 投稿ラボへ                     │
│                                     │
│ 3. 投稿を分析する                    │
│    → 投稿一覧へ                     │
└─────────────────────────────────────┘
```

**条件:**
- 運用計画が未作成の場合のみ表示
- 運用計画作成後は非表示

**実装優先度:** 高

---

### 提案5: 空状態の改善

**実装内容:**
- データがない場合の説明を充実
- 次のアクションを明確に提示
- イラストやアイコンで視覚的に分かりやすく

**例（投稿一覧ページ）:**
```
┌─────────────────────────────────────┐
│         📝                            │
│                                     │
│  まだ投稿がありません                │
│                                     │
│  まずは投稿ラボで投稿文を            │
│  生成してみましょう！                │
│                                     │
│  [投稿ラボへ]                        │
└─────────────────────────────────────┘
```

**実装優先度:** 中

---

### 提案6: ヘルプ・FAQページの追加

**実装内容:**
- サイドバーに「ヘルプ」メニューを追加
- よくある質問（FAQ）を表示
- 各機能の使い方を説明
- 用語集を追加

**実装優先度:** 低（後回しでも可）

---

### 提案7: コンテキストヘルプ（右下のヘルプボタン）

**実装内容:**
- 全ページの右下に固定のヘルプボタンを配置
- クリックで現在のページの説明を表示
- チャット形式のヘルプ（将来的に）

**実装優先度:** 中

---

## 実装優先順位

### Phase 1（最優先）
1. **ホームページに「クイックスタート」セクション** ✅ すぐに効果が出る
2. **各ページに「このページについて」セクション** ✅ 各ページの理解が深まる

### Phase 2（次優先）
3. **初回ログイン時のウェルカムツアー** ✅ 初回体験が向上
4. **空状態の改善** ✅ データがない場合の導線が明確に

### Phase 3（中期的）
5. **ツールチップの拡充** ✅ 詳細な説明が必要な箇所に追加
6. **コンテキストヘルプ** ✅ いつでもヘルプにアクセス可能

### Phase 4（長期的）
7. **ヘルプ・FAQページ** ✅ 包括的なドキュメント

## まとめ

初見ユーザーの使いやすさを向上させるため、以下の改善を提案します：

1. **即効性のある改善**: クイックスタートセクション、ページ説明セクション
2. **初回体験の向上**: ウェルカムツアー、空状態の改善
3. **詳細な説明**: ツールチップの拡充、コンテキストヘルプ

これらの改善により、初見ユーザーが「どこから始めればいいか」「この機能は何をするのか」を理解しやすくなります。
